`$ARGUMENTS`에 대한 조사 결과를 지식 베이스에 저장하세요.

## 절차

### 1. 저장할 내용 파악

사용자가 방금 공유한 조사 결과 또는 대화 내용을 확인한다.
내용이 불명확하면 사용자에게 "저장할 내용을 붙여넣어 주세요"라고 요청한다.

### 2. 적절한 파일 선택

`.claude/knowledge/INDEX.md`를 읽고 `$ARGUMENTS`가 어느 파일에 속하는지 판단한다:

| 내용 유형 | 저장 파일 |
|----------|----------|
| RSS 피드 URL | `rss-feeds.md` |
| 외부 API | `free-apis.md` |
| 코드 패턴, 주의사항 | `job-patterns.md` |
| 위에 해당 없음 | 새 파일 생성 (`{주제}.md`) |

### 3. 형식에 맞게 추가

각 파일의 기존 표 형식을 유지하면서 새 행을 추가한다:

**rss-feeds.md에 추가하는 경우:**
```
| {이름} | `{URL}` | {언어} | ✅ | {오늘날짜} | {비고} |
```

**free-apis.md에 추가하는 경우:**
```
| {이름} | `{엔드포인트}` | {무료한도} | {키필요여부} | ✅ | {오늘날짜} | {비고} |
```

**job-patterns.md에 추가하는 경우:**
패턴 제목(`## 패턴: ...`)과 코드 블록 형식으로 추가한다.

### 4. INDEX.md 업데이트

- 해당 파일의 `항목 수`를 1 증가시킨다.
- `마지막 업데이트`를 오늘 날짜로 변경한다.
- 새 태그가 생기면 `태그 인덱스`에 추가한다.
- 새 파일을 만든 경우 `파일 목록` 표에 행을 추가한다.

### 5. 완료 확인

저장된 내용을 요약해서 보여주고, `/lookup $ARGUMENTS`로 바로 조회할 수 있다고 안내한다.
